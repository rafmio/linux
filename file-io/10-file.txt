Файл

Если файловая позиция находится за пределами файла - промежуточные байты заполняются нулями

Длину файла можно изменить с помощью операции "Усечение" - как удлиннить, так и укоротить.
При удлиннении новые байты заполняются нулями

Внутри ядра Linux все операции файловой системы трактуются именно в контексте блоков. 
Блок - знаменатель всего I/O. Любые операции I/O не могут осущ-ся над объемом инфы, кот не кратен размеру блока.

На внутрисистемном уровне буферизация происходит посредством отложенных записей, объединения  смежных запросов ввода-вывода и опережающего считывания. 
Пользовательская буферизация выполняется иначе.

Указатели файлов. Стандартный ввод-вывод не работает непосредтсвенно с fd, вместо этого - указатели. В библиотеке С указатель на файл ассоциируется с fd (отображается на него).

В терминологии I/O открытый файл называется потоком данных.

При открытии файлов fopen() может присутствовать режим b, который в Linux игнорируется. Linux воспринимает текст и двоичные файлы одинаково. 

fopen() 
- r и r+ - поток данных устанавливается в начало файла
- w и w+ - файл усекается до нулевой величины, если существует. Поток данных  - в начало
- a и a+ - поток уст-ся в конце файла

Файл можно открыть используя и файловый дескриптор fd - функция fdopen()

Закрытие всех потоков данных - fcloseall() - перед закрытием все потоки данных сбрасываются на диск
