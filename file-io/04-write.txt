#include <unistd.h>
ssize_t write(int fd, const void *buffer, size_t count);

Возвращает:
	- количество записанных байтов
	- (-1) при ошибке

	Отличие аттрибутов ф-ции от read(): в read() буффер объявлен как void *buffer, а в write() как const void *buffer, чтобы нельзя было затереть.
	write() может вернуть меньше запрашиваемых на запись байт. Причина - переполнение диска или достиж.огранич.ресурсца процесса на размеры ф-ла (RLIMIT_FSIZE).
	При выполнении IO в отношении дискового файла успешный выход из write() не гарантирует перемещения данных на диск, т.к. ядро занимается буферизацией дискового IO, чтобы сократить объем работы с диском и ускорить выполнение системного вызова write().
