У каждого процесса есть связанный с ним строковый массив , кот.назыв.списком переменных среды (окружения) или просто средой (окружением). Каждая строка имеет вид name=value.
Когда создается новый пр-сс, он наследует копию среды своего родителя. Это простая и часто использ.форма межпроц.в-вия (IPC) - среда представляет способ переноса инфы от род.пр-са дочернему. После создания дочер.пр-са любой проц-с может изменить свою собс.среду.

Переменные среды часто исп-ся оболочкой.
Вывести переменные среды:
$ env
$ printenv

Список переменных сред - в файлах:
- /proc/<PID>/environ
- /proc/self/environ

В Си список переменных среды м.б. доступен с пом. глоб.переменной char **environ. 
Как и argv, environ указывает на список указателей на строки, заканчивающ. нулев.байтами, а сам список заканч. NULL.

Изменение переменных окружения
int putenv(char *string); // добавляет новую переменную к среде вызывающего ее процесса
			  // либо изменяет знач.сущетствующей
			  // после вызова putenv() эта строка становится частью среды

int setenv(const char *name, const char *value, int overwrite); // созд.нов. перем-ю среды
int unsetenv(const char *name); // удаляет из среди переменную
int clearenv(void);	// удаляет среду
